container access

  • Shell into running container
docker attach container_name
You can detach from a container and leave it running using the CTRL-p CTRL-q key sequence
If it won't unattach kill process for attach
  • alternative shell method
    • The docker exec command allows you to run commands inside a Docker container.
$ docker exec -it container_name bash
  • You can copy files,cmds,etc. from the container to the filesystem and back
$ docker cp -L ghost:/usr/src/ghost/config.js ./config.js
$ cp ./config.js /var/lib/ghost/config.js
  • You can mount a filesystem on a container to your host
$ docker run -v "hostFileSystem":"ContainerFileSystem" "containerName"
$ docker run -v /home/test:/usr/test unbuntu
  • Start vs Run docker commands
docker run -d image_name -p 80:2368 container_name - Spinup a new container detached and with port mapping
docker stop container_name
docker start container_name - Start one or more stopped containers
