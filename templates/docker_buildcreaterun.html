docker build / create / run / WTF

Learning docker it may help to understand the difference between these commands. There is a hierarchy to their usage that brings you from a raw image to a working container.
docker Command Reference

  • docker build -t arangodb . 
    •   This builds the image named arangodb from the information in Dockerfile
  • docker create -t -i fedora bash 
    •   The docker create command creates a writeable container layer over the specified image and prepares it for running the specified command.
  The container ID is then printed to STDOUT.
  • docker start -a -i 
    •   You can then use the docker start  command to start the container at any point.
  • docker run --name test -d debian 
    •   docker run creates AND starts the writeable container named "test" from the debian image and detachs so the container runs in the background.

    docker-compose "Building a system from the command line"

    The docker-compose command pulls together containers into a system that performs some function. It reads the default file docker-compose.yml or an alternative.

  • docker-compose Command Reference
  • docker swarm Key Concepts

  • docker-compose build 
    •   This builds the containers defined in the docker-compose.yml file but does not start them.
  • docker-compose up 
    •   This builds the containers defined in the docker-compose.yml and starts them up.
  • docker-compose up --build 
    •   The --build switch forces a rebuild of the images supporting the containers and then containers are created and started.
