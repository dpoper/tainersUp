Volume Mgmt

Docker has a variety of different approaches for data storage. Each has their drawbacks so you decide.

    Storage in a container 
    Storage in a container is quick and dirty however when you delete the container the data is lost.
    Storage to a volume 
    Stores to the host disk for persistence however you are now tied to that host so portability suffers.
